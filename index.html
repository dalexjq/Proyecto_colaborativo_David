Proyecto Colaborativo Entornos De Desarrollo

Una guía sobre cómo usar Git, GitHub y Git Bash para gestionar tu proyecto.

¿Qué es Git?

Git es un sistema de control de versiones distribuido, utilizado para llevar un seguimiento de los cambios en los archivos de un proyecto. Es fundamental en el desarrollo de software, especialmente cuando se trabaja en equipo.

¿Qué es GitHub?

GitHub es una plataforma en línea que utiliza Git para gestionar proyectos de software. Permite a los desarrolladores almacenar y compartir su código con otros, colaborar y llevar un registro completo de las versiones del proyecto.

¿Qué es Git Bash?

Git Bash es una aplicación que emula la línea de comandos de Unix, permitiendo ejecutar comandos Git de forma eficiente en Windows. Es la interfaz que usamos para interactuar con Git de manera rápida.

Pasos para subir un proyecto a GitHub usando Git Bash

  • 1. Inicializar un repositorio Git: Usa el comando git init para crear un repositorio en tu proyecto local.
  • 2. Agregar los archivos: Usa git add . para agregar todos los archivos de tu proyecto al área de preparación.
  • 3. Realizar el primer commit: Usa git commit -m "Primer commit" para registrar el primer cambio.
  • 4. Crear un repositorio en GitHub: Ve a GitHub y crea un nuevo repositorio sin inicializarlo con un README.
  • 5. Conectar tu repositorio local con GitHub: Usa git remote add origin https://github.com/tuusuario/tu-repositorio.git.
  • 6. Subir archivos a GitHub: Usa git push -u origin master para enviar los archivos a GitHub.
